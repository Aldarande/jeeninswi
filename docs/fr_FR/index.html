JeeNinSwi

Contrôle parental Nintendo Switch & Switch 2 pour Jeedom.
Supervision des profils, temps de jeu et blocage à distance.

Présentation

JeeNinSwi intègre l'application Nintendo Switch Parental Controls dans Jeedom. Il vous permet de superviser et de contrôler le temps de jeu de vos enfants directement depuis votre tableau de bord, sans ouvrir l'application Nintendo sur votre téléphone.

Ce que le plugin ne peut pas faire

Limitations liées à l'API Nintendo :

  • Recevoir des événements en temps réel — la mise à jour se fait par polling uniquement
  • Accéder à l'avatar Nintendo (non exposé par l'API Parental Controls)
  • Accéder à la liste d'amis ou aux messages Nintendo

Installation

Prérequis

  • Jeedom 4.4 ou supérieur
  • Python 3.8+ sur la machine Jeedom
  • Accès à l'app Nintendo Switch Parental Controls sur votre téléphone
  • Un compte Nintendo parent associé à la console enfant

Étapes

  1. 1
    Installer le plugin
    Depuis le Market Jeedom ou en copiant le dossier dans plugins/.
  2. 2
    Installer les dépendances Python
    Allez dans Plugins → Gestion des plugins → JeeNinSwi et cliquez sur Installer les dépendances.
    Cette étape crée un environnement virtuel Python isolé dans resources/venv/ et installe pynintendoparental, aiohttp et croniter.
    Pourquoi un venv ? Les dépendances sont isolées du Python système — aucun paquet n'est installé globalement.
  3. 3
    Activer le plugin et démarrer le démon
    Dans la page de configuration, activez le plugin et démarrez le démon. Le démon est un processus Python persistant qui communique avec l'API Nintendo.
  4. 4
    Obtenir un token Nintendo
    Ouvrez la configuration d'un équipement et cliquez sur Assistant Token Nintendo. Voir la section suivante.

Obtenir le token Nintendo

Le plugin utilise un token de session Nintendo pour accéder à l'API Parental Controls. Ce token est obtenu via un processus OAuth en 3 étapes, depuis l'assistant intégré à Jeedom.

⚠️ Compte parent requis Connectez-vous avec le compte Nintendo parent (celui qui administre le contrôle parental), pas avec le compte de l'enfant.
  1. 1
    Générer l'URL de connexion
    Dans la configuration d'un équipement, cliquez sur Assistant Token Nintendo puis Générer l'URL de connexion. Une URL Nintendo apparaît — copiez-la et ouvrez-la dans votre navigateur.
  2. 2
    Se connecter avec son compte Nintendo
    Connectez-vous sur le site Nintendo. La page va tenter de vous rediriger vers l'URL npf54789befb391a838://auth#session_token_code=… (l'app Parental Controls sur mobile, ou une erreur sur PC). Copiez cette URL complète depuis la barre d'adresse.
    💡 Astuce mobile Sur téléphone, l'URL s'affiche brièvement avant l'ouverture de l'app. Sur PC, le navigateur affiche une erreur "impossible d'ouvrir" mais l'URL est visible dans la barre d'adresse.
  3. 3
    Valider et créer les équipements
    Collez l'URL copiée dans le champ URL de redirection et cliquez sur Valider. Le plugin récupère automatiquement la liste de vos consoles et crée les équipements Jeedom correspondants.

Configuration des équipements

Chaque console Nintendo apparaît comme un équipement Jeedom indépendant. Si vous relancez l'assistant token, les équipements existants ne sont pas recréés — seul le token est mis à jour.

ℹ️ Nom de l'équipement Le nom de l'équipement dans Jeedom est indépendant du profil Nintendo. Vous pouvez le renommer librement sans impact sur l'app Nintendo, et vice versa.

Dépannage

Le démon ne démarre pas
  • Vérifiez que les dépendances sont installées (état OK dans la page configuration)
  • Consultez les logs : Analyse → Logs → jeeninswi
  • Relancez l'installation des dépendances si nécessaire
  • Vérifiez que Python 3.8+ est disponible : python3 --version
Les données ne se mettent pas à jour
  • Cliquez sur ↺ dans le widget pour forcer un rafraîchissement
  • Consultez les logs pour vérifier la communication avec l'API Nintendo
  • Si le token est expiré (plusieurs mois), relancez l'Assistant Token Nintendo
L'assistant token affiche une erreur
  • "URL de redirection invalide" : L'URL copiée ne commence pas par npf54789befb391a838://auth#. Copiez l'URL complète depuis la barre d'adresse.
  • "Impossible de générer l'URL" : Dépendances Python non installées. Relancez l'installation des dépendances.
  • "Fichier d'état introuvable" : L'état OAuth a expiré. Relancez depuis l'étape 1 de l'assistant.
Dépendances en erreur (état NOK)
  • Vérifiez les logs de dépendances : Analyse → Logs → jeeninswi_dependancy
  • Sur Debian/Ubuntu, installez le paquet venv si manquant : apt-get install -y python3-venv
  • Relancez l'installation des dépendances depuis la page de gestion du plugin

Architecture technique

Pour les développeurs ou utilisateurs avancés :

Jeedom (PHP) Démon Python API Nintendo │ │ │ │── sendToDaemon() ──────▶│ │ │ POST /action │── NintendoParental ──▶│ │ │ (pynintendoparental)│ │◀── callback.php ──────│◀─────────────────────│ │ POST + JSON data │ │ │── updateFromData() ──▶│ │ │ (cmd::event) │ │

Composants

FichierRôle
core/class/jeeninswi.class.phpLogique métier Jeedom (eqLogic, cmd, toHtml, callback)
core/ajax/jeeninswi.ajax.phpEndpoints AJAX (assistant token, statut, actions)
core/php/callback.phpRéception HTTP des données du démon
resources/jeeninswid/jeeninswid.pyDémon Python — poll Nintendo, serveur HTTP aiohttp
